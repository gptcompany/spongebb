### Architecture

```
NewsAlertEngine
├── __init__(discord_client, keywords=BREAKING_KEYWORDS)
├── check_and_alert(news_item) → bool  # Returns True if alerted
├── _match_keywords(text) → list[tuple[keyword, config]]
├── _should_alert(matches, source) → bool
├── _create_alert_embed(news_item, matches) → DiscordEmbed
└── _get_priority(matches) → str

Integration with existing alerts:
├── Reuses src/liquidity/alerts/discord.py (DiscordClient)
├── New alert_type: "breaking_news"
└── Rate limit: 30 seconds per source
```

### Key Decisions

- **Keyword matching**: Case-insensitive, partial match
- **Priority hierarchy**: Critical > High > Medium > Low
- **Source filtering**: Keywords apply only to relevant CBs
- **Rate limiting**: Reuse existing DiscordClient rate limiter
- **Deduplication**: Track alerted GUIDs to prevent repeats
