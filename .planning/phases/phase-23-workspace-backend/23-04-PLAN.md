### Task 6: Create test_cf_access.py ŌĆö Cloudflare Access Middleware Tests

**File:** `tests/unit/test_workspace/test_cf_access.py`
**Action:** create

Tests for the Cloudflare Access middleware that checks `Cf-Access-Authenticated-User-Email` header when `LIQUIDITY_CF_ACCESS_ENABLED=true`.

**IMPORTANT ŌĆö Middleware immutability:** Starlette/FastAPI middleware stack is frozen after app instantiation. The workspace_app.py registers CF Access middleware at import time based on `cf_access_enabled` setting. Therefore, tests CANNOT toggle middleware via monkeypatch on a running app.

**Testing strategy ŌĆö app factory pattern:**
Create a helper function that builds a fresh FastAPI app with or without middleware:

```python
def create_test_app(cf_access_enabled: bool = False) -> FastAPI:
    """Create a fresh app instance for testing CF Access middleware."""
    from liquidity.api.server import app as base_app
    from fastapi.testclient import TestClient

    if cf_access_enabled:
        from liquidity.api.middleware import CloudflareAccessMiddleware
        base_app.add_middleware(CloudflareAccessMiddleware)

    return base_app
```

Or more robustly, test the middleware class in isolation:

```python
from starlette.testclient import TestClient
from starlette.applications import Starlette
from starlette.routing import Route
from starlette.responses import PlainTextResponse
from liquidity.api.middleware import CloudflareAccessMiddleware

def homepage(request):
    return PlainTextResponse("OK")

# Build minimal app with middleware for testing
test_app_with_cf = Starlette(routes=[Route("/test", homepage), Route("/health", homepage)])
test_app_with_cf.add_middleware(CloudflareAccessMiddleware)
```

This approach tests the middleware behavior in isolation without needing to toggle it on the real app.

**Tests:**
- `test_cf_access_middleware_blocks_without_header` ŌĆö Minimal Starlette app with middleware: GET `/test` without CF header returns 403
- `test_cf_access_middleware_allows_with_header` ŌĆö GET `/test` with `Cf-Access-Authenticated-User-Email: user@example.com` returns 200
- `test_cf_access_middleware_exempts_health` ŌĆö GET `/health` without CF header returns 200 (exempt path)

Total: 3 tests.
