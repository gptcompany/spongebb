# Plan 24-01: Refresh Intervals & Stale Time Optimization

## Goal
Align `refetchInterval` and `staleTime` values across all 18 widget endpoints to match actual data source update frequencies, reducing unnecessary API calls while keeping data fresh.

## Rationale
Current state:
- Metric/chart endpoints have staleTime, table endpoints don't
- Some table endpoints use 15m refetch for weekly/monthly data (wasteful)
- No endpoint distinguishes between real-time and quarterly data

**Note on table vs metric intervals:** Table endpoints (detail views) use longer
refetch intervals than metric KPI cards (same data). This is intentional:
KPI cards are always-visible dashboard widgets where users expect responsiveness.
Table endpoints are drill-down views opened on demand. Both serve from the same
cache, so the KPI's shorter interval keeps the cache warm for both.
