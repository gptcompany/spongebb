# v5.0 OpenBB Platform Integration

**Status:** Active
**Phases:** 23-25 (10 plans total: 7 firm + 3 conditional)
**Started:** 2026-02-21

## Goal

Elevate the liquidity monitor from standalone Dash app to native OpenBB ecosystem component. Expose all 14 FastAPI endpoints as OpenBB Workspace widgets, optimize widget UX, and optionally create a native provider extension for Python SDK access.

## Strategy

B (Workspace backend) → Polish → A (Provider extension, conditional)

Research concluded that the `openbb-api --app` path is the correct primary integration. Provider extension (`obb.liquidity.*`) is optional Phase 3 with an explicit gate (Issue #7113 + demand).
