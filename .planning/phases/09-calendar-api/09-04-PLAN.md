# Plan 09-04: Docker Deployment

## Objective
Create Docker container for the API server with proper configuration and compose setup.

## Requirements Addressed
- User preference: Docker deployment
- Production-ready container with health checks

## Tasks

### 1. Create Dockerfile
```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install uv for faster package management
RUN pip install uv

# Copy dependency files first (cache layer)
COPY pyproject.toml uv.lock ./

# Install dependencies
RUN uv sync --frozen --no-dev

# Copy application code
COPY src/ ./src/

# Expose port
EXPOSE 8000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# Run with uvicorn
CMD ["uv", "run", "uvicorn", "liquidity.api:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2. Create docker-compose.yml
```yaml
# docker-compose.yml
version: "3.8"

services:
  liquidity-api:
    build: .
    container_name: liquidity-api
    restart: unless-stopped
    ports:
      - "8000:8000"
    environment:
      - LIQUIDITY_QUESTDB_HOST=${QUESTDB_HOST:-questdb}
      - LIQUIDITY_QUESTDB_PORT=${QUESTDB_PORT:-8812}
      - LIQUIDITY_FRED_API_KEY=${FRED_API_KEY}
      - LIQUIDITY_LOG_LEVEL=${LOG_LEVEL:-INFO}
    depends_on:
      questdb:
        condition: service_healthy
    networks:
      - liquidity-net
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 10s

  questdb:
    image: questdb/questdb:8.0.0
    container_name: questdb
    restart: unless-stopped
    ports:
      - "9000:9000"  # Web console
      - "8812:8812"  # PostgreSQL wire protocol
      - "9009:9009"  # InfluxDB line protocol
    volumes:
      - questdb-data:/var/lib/questdb
    networks:
      - liquidity-net
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  questdb-data:

networks:
  liquidity-net:
    driver: bridge
```

### 4. Create docker-compose.override.yml (Development)
```yaml
# docker-compose.override.yml (gitignored, for local dev)
version: "3.8"

services:
  liquidity-api:
    build:
      context: .
      dockerfile: Dockerfile
    volumes:
      - ./src:/app/src:ro  # Hot reload in dev
    command: ["uv", "run", "uvicorn", "liquidity.api:app", "--host", "0.0.0.0", "--port", "8000", "--reload"]
```

### 5. Environment Template
```bash
# .env.example
QUESTDB_HOST=questdb
QUESTDB_PORT=8812
FRED_API_KEY=your_fred_api_key
LOG_LEVEL=INFO
```

### 6. Update pyproject.toml Scripts
```toml
[project.scripts]
liquidity-api = "liquidity.api:main"

[tool.uv.scripts]
serve = "uvicorn liquidity.api:app --host 0.0.0.0 --port 8000"
serve-dev = "uvicorn liquidity.api:app --host 0.0.0.0 --port 8000 --reload"
```

### 7. Add main() Entry Point (api/__init__.py)
```python
import uvicorn
from .server import app

def main():
    uvicorn.run(app, host="0.0.0.0", port=8000)

__all__ = ["app", "main"]
```

### 8. Test Docker Build
```bash
# Build
docker compose build

# Run
docker compose up -d

# Check logs
docker compose logs -f liquidity-api

# Test health
curl http://localhost:8000/health

# Stop
docker compose down
```
