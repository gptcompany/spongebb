# Plan 23-02: Workspace Metric + Chart Endpoints

## Goal
Create dedicated `/workspace/metrics/*` and `/workspace/charts/*` endpoints returning widget-compatible response formats (metric JSON and Plotly JSON) that OpenBB Workspace can render natively as KPI cards and interactive charts.

## Requirements Covered
- **WS-04**: Dedicated `/workspace/metrics/*` endpoints returning `MetricResponseModel`-compatible JSON for metric widgets (Net Liquidity value + WoW delta, Global Liquidity total, Stealth QE score, Regime badge)
- **WS-05**: Dedicated `/workspace/charts/*` endpoints returning Plotly JSON for chart widgets (Net Liquidity time series, Global Liquidity breakdown)

## Context
OpenBB Workspace expects specific response shapes that differ from the existing REST endpoints:
- **Metric widgets** need `{"value": N, "label": "...", "delta": D}` format -- simple KPI cards
- **Chart widgets** need Plotly JSON via `.to_plotly_json()` -- Workspace renders Plotly natively
- The existing `/liquidity/net`, `/regime/current`, etc. return full Pydantic models (e.g., `NetLiquidityResponse` with walcl/tga/rrp breakdown) designed for machine consumption. These shapes are incompatible with metric widgets.
- Per CONTEXT.md: "Zero duplication of business logic -- workspace routes wrap existing calculators"
- Per research SUMMARY.md: "Dedicated `/workspace/*` routes for shape-divergent responses... Add `openapi_extra` to existing table endpoints (compatible), but add dedicated `/workspace/*` endpoints for metric and chart widgets to avoid breaking existing consumers."
- All endpoints are GET-only. Each gets `openapi_extra` with `widget_config` metadata for Workspace widget discovery.

### Calculator Interfaces Used
- `NetLiquidityCalculator.get_current()` -> `NetLiquidityResult` (net_liquidity, weekly_delta, sentiment, walcl, tga, rrp)
- `NetLiquidityCalculator.calculate(start_date, end_date)` -> DataFrame (timestamp, net_liquidity, walcl, tga, rrp)
- `GlobalLiquidityCalculator.get_current(tier=1)` -> `GlobalLiquidityResult` (total_usd, weekly_delta, fed_usd, ecb_usd, boj_usd, pboc_usd)
- `GlobalLiquidityCalculator.calculate(start_date, end_date, tier=1)` -> DataFrame (timestamp, global_liquidity, fed_usd, ecb_usd, boj_usd, pboc_usd)
- `StealthQECalculator.get_current()` -> `StealthQEResult` (score_daily, status, rrp_level, tga_level)
- `RegimeClassifier.classify()` -> result with (direction.value, intensity, confidence)

### Existing Dependency Injection (from deps.py)
- `NetLiquidityCalcDep` = `Annotated[NetLiquidityCalculator, Depends(get_net_liquidity_calculator)]`
- `GlobalLiquidityCalcDep` = `Annotated[GlobalLiquidityCalculator, Depends(get_global_liquidity_calculator)]`
- `StealthQECalcDep` = `Annotated[StealthQECalculator, Depends(get_stealth_qe_calculator)]`
- `RegimeClassifierDep` = `Annotated[RegimeClassifier, Depends(get_regime_classifier)]`

## Tasks

### Task 1: Create workspace response schemas
**File:** `src/liquidity/api/workspace_schemas.py`
**Action:** create

Create a Pydantic BaseModel for workspace metric responses. This is a simplified schema -- NOT the full `MetricResponseModel` from `openbb-platform-api` (which may have extra fields we don't need). Keep it minimal and compatible with the Workspace metric widget expected shape.

```python
from pydantic import BaseModel, Field

class WorkspaceMetric(BaseModel):
    """Response model for OpenBB Workspace metric widgets.

    Compatible with MetricResponseModel expected shape:
    {"value": N, "label": "...", "delta": D}
    """
    value: float = Field(description="Primary metric value")
    label: str = Field(description="Human-readable label for the metric")
    delta: float | None = Field(default=None, description="Change value (e.g., WoW delta)")
    unit: str | None = Field(default=None, description="Unit label (e.g., '$B', '%', 'score')")
    sentiment: str | None = Field(default=None, description="Sentiment/status indicator")
```

This model is intentionally flat and simple. No nested objects, no metadata wrapper. Workspace metric widgets render `value` as the headline number, `label` as the title, and `delta` as the change indicator.

### Task 2: Create workspace_routes.py with metric endpoints
**File:** `src/liquidity/api/workspace_routes.py`
**Action:** create

Create `workspace_router = APIRouter(prefix="/workspace", tags=["workspace"])` with 4 metric endpoints. Each endpoint:
- Uses existing dependency injection from `deps.py` (the same calculator singletons)
- Returns `WorkspaceMetric` response
- Has `openapi_extra` with `widget_config` including `type: "metric"`, `category`, `subCategory`, `refetchInterval`
- Wraps calculator calls in try/except with HTTPException (same pattern as existing routers)

**Metric endpoints (all GET):**

1. **`GET /workspace/metrics/net-liquidity`**
   - Calls `calculator.get_current()` via `NetLiquidityCalcDep`
   - Returns: `WorkspaceMetric(value=result.net_liquidity, label="Net Liquidity", delta=result.weekly_delta, unit="$B", sentiment=result.sentiment.value)`
   - `openapi_extra.widget_config`: `{"name": "Net Liquidity", "description": "Hayes formula: WALCL - TGA - RRP (billions USD)", "type": "metric", "category": "Macro Liquidity", "subCategory": "Fed", "refetchInterval": 900000, "staleTime": 300000, "gridData": {"w": 10, "h": 4}}`

2. **`GET /workspace/metrics/global-liquidity`**
   - Calls `calculator.get_current(tier=1)` via `GlobalLiquidityCalcDep`
   - Returns: `WorkspaceMetric(value=result.total_usd, label="Global Liquidity", delta=result.weekly_delta, unit="$B")`
   - `openapi_extra.widget_config`: `{"name": "Global Liquidity", "description": "Fed + ECB + BoJ + PBoC aggregate (billions USD)", "type": "metric", "category": "Macro Liquidity", "subCategory": "Global", "refetchInterval": 900000, "staleTime": 300000, "gridData": {"w": 10, "h": 4}}`

3. **`GET /workspace/metrics/stealth-qe`**
   - Calls `calculator.get_current()` via `StealthQECalcDep`
   - Returns: `WorkspaceMetric(value=result.score_daily, label="Stealth QE Score", delta=None, unit="score", sentiment=result.status)`
   - `openapi_extra.widget_config`: `{"name": "Stealth QE Score", "description": "Hidden liquidity injection detection (0-100)", "type": "metric", "category": "Macro Liquidity", "subCategory": "Stealth", "refetchInterval": 900000, "staleTime": 300000, "gridData": {"w": 10, "h": 4}}`

4. **`GET /workspace/metrics/regime`**
   - Calls `classifier.classify()` via `RegimeClassifierDep`
   - Returns: `WorkspaceMetric(value=result.intensity, label=f"Regime: {result.direction.value}", delta=None, unit="intensity", sentiment=result.direction.value)`
   - The label embeds the regime direction (e.g., "Regime: EXPANSION") so the metric card displays the regime badge clearly
   - `openapi_extra.widget_config`: `{"name": "Liquidity Regime", "description": "EXPANSION or CONTRACTION with intensity (0-100)", "type": "metric", "category": "Macro Liquidity", "subCategory": "Regime", "refetchInterval": 900000, "staleTime": 300000, "gridData": {"w": 10, "h": 4}}`

Error handling pattern (same as existing routers):
```python
try:
    result = await calculator.get_current()
    return WorkspaceMetric(...)
except ValueError as e:
    raise HTTPException(status_code=503, detail=str(e))
except Exception as e:
    logger.exception("Unexpected error in workspace metric endpoint")
    raise HTTPException(status_code=500, detail=str(e))
```

### Task 3: Add chart endpoints to workspace_routes.py
**File:** `src/liquidity/api/workspace_routes.py`
**Action:** modify (append to existing file)

Add 2 chart endpoints to the same `workspace_router`. Chart endpoints return Plotly JSON (`dict`) from `.to_plotly_json()`. Import `plotly.graph_objects as go` at the top of the file.

1. **`GET /workspace/charts/net-liquidity`**
   - Calls `calculator.calculate(start_date, end_date)` via `NetLiquidityCalcDep`
   - `start_date` defaults to 2 years ago (`datetime.now(UTC) - timedelta(days=730)`)
   - Creates a Plotly figure: `go.Figure()` with a single `go.Scatter` trace (x=df["timestamp"], y=df["net_liquidity"], mode="lines", name="Net Liquidity ($B)")
   - Layout: `title="Fed Net Liquidity Index"`, `yaxis_title="Billions USD"`, `template="plotly_dark"`, `hovermode="x unified"`
   - Returns `fig.to_plotly_json()` (returns a dict, which FastAPI serializes as JSON)
   - Return type annotation: `dict` (not a Pydantic model -- Plotly JSON is opaque to FastAPI)
   - `openapi_extra.widget_config`: `{"name": "Net Liquidity Chart", "description": "Fed Net Liquidity (WALCL - TGA - RRP) time series", "type": "chart", "category": "Macro Liquidity", "subCategory": "Fed", "refetchInterval": 3600000, "staleTime": 1800000, "gridData": {"w": 20, "h": 8}}`

2. **`GET /workspace/charts/global-liquidity`**
   - Calls `calculator.calculate(start_date, end_date, tier=1)` via `GlobalLiquidityCalcDep`
   - `start_date` defaults to 2 years ago
   - Creates a Plotly figure with stacked area traces for each CB component:
     - `go.Scatter(x=df["timestamp"], y=df["fed_usd"], name="Fed", stackgroup="one", mode="lines")`
     - `go.Scatter(x=df["timestamp"], y=df["ecb_usd"], name="ECB", stackgroup="one", mode="lines")`
     - `go.Scatter(x=df["timestamp"], y=df["boj_usd"], name="BoJ", stackgroup="one", mode="lines")`
     - `go.Scatter(x=df["timestamp"], y=df["pboc_usd"], name="PBoC", stackgroup="one", mode="lines")`
   - Layout: `title="Global Liquidity Breakdown"`, `yaxis_title="Billions USD"`, `template="plotly_dark"`, `hovermode="x unified"`
   - Returns `fig.to_plotly_json()`
   - `openapi_extra.widget_config`: `{"name": "Global Liquidity Chart", "description": "Central bank balance sheet breakdown (stacked area)", "type": "chart", "category": "Macro Liquidity", "subCategory": "Global", "refetchInterval": 3600000, "staleTime": 1800000, "gridData": {"w": 20, "h": 8}}`

Both chart endpoints handle empty DataFrames gracefully: if `df.empty`, return an empty Plotly figure with an annotation "No data available" instead of raising an exception.

### Task 4: Register workspace router in workspace_app.py
**File:** `src/liquidity/openbb_ext/workspace_app.py`
**Action:** modify

This file will be created by Plan 23-01 (it imports `app` from `server.py` and re-exports it). This task adds the workspace router registration.

After the existing `app` import, add:
```python
from liquidity.api.workspace_routes import workspace_router
app.include_router(workspace_router)
```

This must be placed AFTER the `app` import and BEFORE the module-level export, so that `openbb-api --app workspace_app:app` sees all workspace routes in the OpenAPI schema (and thus in auto-generated `widgets.json`).

**Important:** If Plan 23-01 has not yet created `workspace_app.py`, this task creates it with the minimal content:
```python
"""OpenBB Workspace app entry point.

Imports the existing FastAPI app from server.py, adds workspace-specific
routes, and re-exports for `openbb-api --app workspace_app:app`.
"""
from liquidity.api.server import app  # noqa: F401
from liquidity.api.workspace_routes import workspace_router

app.include_router(workspace_router)
```

Also ensure `src/liquidity/openbb_ext/__init__.py` exists (create empty if not).
