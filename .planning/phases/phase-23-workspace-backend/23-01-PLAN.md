# Plan 23-01: Foundation — openbb-platform-api dependency, version pin, CORS, workspace_app.py

## Goal

Set up `openbb-platform-api` dependency, pin OpenBB version to `>=4.4.0,<4.7.0`, configure explicit CORS origins for Workspace + Cloudflare Access, add optional CF Access header verification middleware, and create `workspace_app.py` that assembles the full app for `openbb-api --app` launch.

## Requirements Covered

- **WS-01** (partial): Workspace backend foundation — app assembly point created, launch command ready
- **WS-02**: CORS configuration for `pro.openbb.co`, `localhost:1420`, `liquidity.princyx.xyz`
- **WS-03**: `openbb-platform-api` dependency added, `openbb-api --app` launch path established
- **VS-01**: OpenBB version pinned to `>=4.4.0,<4.7.0` to prevent silent breakage from endpoint renames

## Context

Research (`.planning/research/SUMMARY.md`) confirms the primary integration path is `openbb-api --app`, wrapping the existing FastAPI app. The project currently has `openbb>=4.0.0` (too permissive — v4.4.0 to v4.5.0 renamed 7+ endpoints). CORS is wildcard `allow_origins=["*"]` which works but is insecure for production Workspace use. The `openbb_ext/` directory does not exist yet — this plan creates the foundation.

Current installed versions: openbb 4.6.0, openbb-platform-api 1.2.3, openbb-core 1.5.8, Python 3.11.

CONTEXT.md locked decisions:
- Desktop App + Browser + Remote targets
- CORS origins: `https://pro.openbb.co`, `http://localhost:1420`, `https://liquidity.princyx.xyz`
- CF Access + header verification (defense in depth), no X-API-KEY middleware
- Single enhanced service approach (workspace routes added to same app)

### Task 4: Create Cloudflare Access header verification middleware

**File:** `/media/sam/1TB/openbb_liquidity/src/liquidity/api/middleware.py`
**Action:** create

Create a Starlette middleware that provides defense-in-depth when behind Cloudflare Access:

```python
"""Cloudflare Access header verification middleware.

Defense-in-depth: CF Access handles primary auth at the edge.
This middleware verifies the Cf-Access-Authenticated-User-Email header
exists when LIQUIDITY_CF_ACCESS_ENABLED=true, preventing bypass if
tunnel misconfigured or internal network compromised.

Enable via: LIQUIDITY_CF_ACCESS_ENABLED=true
"""
import logging

from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import JSONResponse

logger = logging.getLogger(__name__)

# Path prefixes that bypass CF Access verification (health checks, docs, root)
EXEMPT_PATH_PREFIXES = ("/health", "/docs", "/openapi.json", "/redoc", "/widgets.json")


class CloudflareAccessMiddleware(BaseHTTPMiddleware):
    """Verify Cloudflare Access authenticated user header.

    When enabled, rejects requests missing the
    Cf-Access-Authenticated-User-Email header with 403.
    Health, docs, and widget discovery endpoints are exempt.
    Uses prefix matching to handle sub-paths (e.g. /docs/ redirects).
    """

    async def dispatch(self, request: Request, call_next):
        path = request.url.path
        if path == "/" or path.startswith(EXEMPT_PATH_PREFIXES):
            return await call_next(request)

        cf_email = request.headers.get("Cf-Access-Authenticated-User-Email")
        if not cf_email:
            logger.warning(
                "CF Access: missing authenticated user header for %s %s from %s",
                request.method,
                request.url.path,
                request.client.host if request.client else "unknown",
            )
            return JSONResponse(
                status_code=403,
                content={"error": "Cloudflare Access authentication required"},
            )

        return await call_next(request)
```

This middleware is only added to the app when `cf_access_enabled=True` (wired in Task 5).

Launch: openbb-api --app liquidity.openbb_ext.workspace_app:app --host 0.0.0.0 --port 6900

from liquidity.api.server import app  # noqa: F401

_settings = get_settings()

    from liquidity.api.middleware import CloudflareAccessMiddleware

    app.add_middleware(CloudflareAccessMiddleware)
