  <action>
Create TICCollector following existing collector patterns (see fx.py, sofr.py).

**Data sources:**
- Primary: US Treasury CSV files (no auth required)
  - MFH table: https://www.treasury.gov/resource-center/data-chart-center/tic/Documents/slt_table5.txt
  - Holdings: https://www.treasury.gov/resource-center/data-chart-center/tic/Documents/slt_table3.txt
- Fallback: FRED quarterly series (BOGZ1FL263061130Q for official, BOGZ1FL263061145Q for private)

**Key implementation:**
- Use httpx async client for Treasury CSV fetches
- Parse tab-delimited TXT files with pandas
- Implement 2-tier fallback: Treasury direct -> FRED quarterly
- Handle series break at Feb 2023 (Form S -> Form SLT)

**Methods to implement:**
- `collect_major_holders()` -> DataFrame with top 25 foreign holders
- `collect_treasury_holdings()` -> DataFrame with holdings by country
- `collect_aggregate()` -> FRED quarterly aggregate (fallback)
- `collect()` -> Main entry point, returns major holders by default

**Output format (standard):**
- timestamp, series_id, source, value, unit
- series_id examples: "tic_japan_holdings", "tic_china_holdings", "tic_total_foreign"
- unit: "millions_usd"

Register as "tic" in registry.
