## Objective
Implement calendar tracking for liquidity-impacting events: Treasury auctions, CB meetings, tax dates, holidays.

- CAL-02: Month-end/quarter-end liquidity windows

- CAL-04: Fed meeting dates and blackout periods

### 2. Base Classes (base.py)
```python
from dataclasses import dataclass
from datetime import date
from enum import Enum

class EventType(Enum):
    TREASURY_AUCTION = "treasury_auction"
    FED_MEETING = "fed_meeting"
    ECB_MEETING = "ecb_meeting"
    BOJ_MEETING = "boj_meeting"
    BOE_MEETING = "boe_meeting"
    TAX_DATE = "tax_date"
    QUARTER_END = "quarter_end"
    MONTH_END = "month_end"
    HOLIDAY = "holiday"

@dataclass
class CalendarEvent:
    date: date
    event_type: EventType
    title: str
    description: str | None = None
    settlement_date: date | None = None  # For auctions
    impact: str = "medium"  # low/medium/high
```

### 3. Treasury Auction Calendar (treasury.py)
- Fetch from Treasury Direct API (JSON endpoint)
- Track: Bills, Notes, Bonds, TIPS, FRN
- Include settlement dates (T+1 or T+2)
- Impact: high for 10Y/30Y auctions

### 4. Central Bank Meetings (central_banks.py)
- Fed: FOMC schedule (8 per year)
- ECB: Governing Council (6 per year)
- BoJ: MPM (8 per year)
- BoE: MPC (8 per year)
- Include blackout periods (Fed: 10 days before)

### 5. Tax Dates (tax_dates.py)
- Quarterly estimated payments: Jan 15, April 15, June 15, Sept 15
- Corporate tax dates
- Impact: high (TGA spikes)

### 6. US Market Holidays (holidays.py)
- Use `holidays` package for NYSE calendar
- Impact: medium (reduced liquidity)

### 7. Calendar Registry (registry.py)
- Unified interface to query all calendars
- Filter by date range, event type, impact level
- Sort by date
