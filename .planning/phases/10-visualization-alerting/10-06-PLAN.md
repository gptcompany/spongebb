# Plan 10-06: Dashboard Visual Regression (Playwright + CI)

## Goal

Aggiungere una verifica visuale automatica e ripetibile della dashboard, utilizzabile:
- in locale (developer workflow),
- in CI (gating su PR/push),
- in modalità interattiva via Playwright MCP per debug visuale guidato.

## Requirements

| ID | Requirement |
|----|-------------|
| VIZ-09 | Visual regression automatica su dashboard (snapshot-based) |
| OPS-04 | Artifact CI (Playwright report + test-results) su failure |
| DEV-07 | Modalità deterministica, indipendente da dati live/API keys |
| TOOL-02 | Workflow operativo Playwright MCP per ispezione browser |

## Scope

### In Scope
- Config Playwright per esecuzione desktop/mobile Chromium
- Test visuale baseline ("above-the-fold")
- Workflow GitHub Actions dedicato alla regressione visuale
- Baseline snapshots versionati
- Modalità fallback deterministica nel dashboard (`FORCE_FALLBACK` + timestamp fisso)
- Documentazione comandi local/CI

### Out of Scope
- Copertura screenshot di tutti i pannelli dashboard
- Cross-browser parity (Firefox/WebKit)
- Golden snapshots multi-OS

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Test runner | `@playwright/test` | Standard de-facto per visual E2E web |
| Determinismo | Env flags su callback dashboard | Riduce flakiness da clock e dati live |
| Scope snapshot | Above-the-fold | Segnale alto, maintenance contenuta |
| Browser | Chromium desktop + mobile | Copertura iniziale utile con costo basso |
| CI install | `npm ci` + `npx playwright install --with-deps chromium` | Build riproducibile su runner GitHub |
| Debug interattivo | Playwright MCP (config globale Codex) | Ispezione rapida di regressioni UI |
