<task type="auto">
  <name>Task 2: Implement RegimeClassifier</name>
  <files>src/liquidity/analyzers/regime_classifier.py</files>
  <action>
    Create RegimeClassifier class following StealthQECalculator pattern:

    **Enum: RegimeDirection**
    - EXPANSION = "EXPANSION"
    - CONTRACTION = "CONTRACTION"
    (No NEUTRAL - per CONTEXT.md, "neutral" is a cop-out)

    **Dataclass: RegimeResult**
    - timestamp: datetime
    - direction: RegimeDirection (EXPANSION or CONTRACTION)
    - intensity: float (0-100)
    - confidence: str (HIGH/MEDIUM/LOW based on component agreement)
    - net_liq_percentile: float (rolling percentile)
    - global_liq_percentile: float (rolling percentile)
    - stealth_qe_score: float (direct from StealthQEResult)
    - components: str (e.g., "NetLiq:40% Global:40% StealthQE:20%")

    **Class: RegimeClassifier**
    - __init__(self, settings: Settings | None = None)
    - async calculate(self, lookback: int = 90) -> RegimeResult

    **Algorithm (from RESEARCH.md):**
    1. Fetch latest NetLiquidityResult, GlobalLiquidityResult, StealthQEResult
    2. Calculate rolling percentiles (lookback window) for net/global liquidity
    3. Normalize stealth_qe_score to 0-1 scale (score_daily / 100)
    4. Compute weighted composite: net_pct * 0.40 + global_pct * 0.40 + stealth_pct * 0.20
    5. Direction: EXPANSION if composite > 0.5, else CONTRACTION
    6. Intensity: abs(composite - 0.5) * 200 (scale 0-100)
    7. Confidence: HIGH if all 3 components agree on direction, MEDIUM if 2 agree, LOW if split

    **Config (make configurable):**
    - WEIGHT_NET_LIQUIDITY = 0.40
    - WEIGHT_GLOBAL_LIQUIDITY = 0.40
    - WEIGHT_STEALTH_QE = 0.20
    - DEFAULT_LOOKBACK = 90 (days for percentile calculation)

    **Anti-patterns to avoid:**
    - DON'T use fixed thresholds (e.g., $7T = expansion). Use percentiles for adaptivity.
    - DON'T use `rank(pct=True)` on full series (lookahead bias). Use `rolling(lookback).rank()` or `expanding().rank()`.
    - DON'T hardcode weights without config - make them tunable.

    Use logging (logger = logging.getLogger(__name__))
  </action>
